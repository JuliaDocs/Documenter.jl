\section{Basic Markdown}



Documenter can render all the \href{https://docs.julialang.org/en/v1/stdlib/Markdown/}{Markdown syntax supported by the Julia Markdown parser}. You can use all the usual markdown syntax, such as \textbf{bold text} and \emph{italic text} and \texttt{print({\textquotedbl}inline code{\textquotedbl})}.



\hypertarget{9246544299434150790}{}


\section{Code blocks}



Code blocks are rendered as follows:




\begin{lstlisting}
This is an non-highlighted code block.
... Rendered in monospace.
\end{lstlisting}



When the language is specified for the block, e.g. by starting the block with \texttt{```julia}, the contents gets highlighted appropriately (for the language that are supported by the highlighter).




\begin{minted}{julia}
function foo(x::Integer)
    @show x + 1
end
\end{minted}

\section{Mathematics}



For mathematics, both inline and display equations are available. Inline equations should be written as LaTeX between two backticks, e.g. \texttt{``A x{\textasciicircum}2 + B x + C = 0``}. It will render as \(A x^2 + B x + C = 0\).



The LaTeX for display equations must be wrapped in a \texttt{```math} code block and will render like



\begin{equation*}
\begin{split}x_{1,2} = \frac{-B \pm \sqrt{B^2 - 4 A C}}{2A}\end{split}\end{equation*}


By default, the HTML output renders equations with \href{https://katex.org/}{KaTeX}, but \href{https://www.mathjax.org/}{MathJax} can optionally be used as well.



\begin{quote}
\textbf{Warning}

Similar to LaTeX, using \texttt{\$} and \texttt{\$\$} to escape inline and display equations also works. However, doing so is deprecated and this functionality may be removed in a future release.

\end{quote}

\section{Images}



Include images using basic Markdown syntax:



\begin{figure}
\centering
\includegraphics[max width=\linewidth]{logo.png}
\caption{Enter a descriptive caption for the image}
\end{figure}




The path should be relative to the directory of the current file. Alternatively, use \texttt{./} to begin a path relative to the \texttt{src} of the documents, e.g., \texttt{./logo.png}.

\section{Admonitions}



Admonitions are colorful boxes used to highlight parts of the documentation.



Each admonition begins with three \texttt{!!!}, and then the content is indented underneath with four spaces:




\begin{lstlisting}
!!! note "An optional title"
    Here is something you should pay attention to.
\end{lstlisting}



Documenter supports a range of admonition types for different circumstances.



\hypertarget{8401296361138166423}{}


\subparagraph{Note admonition}



\begin{quote}
\textbf{{\textquotesingle}note{\textquotesingle} admonition}

Admonitions look like this. This is a \texttt{!!! note}-type admonition.

Note that admonitions themselves can contain other block-level elements too, such as code blocks. E.g.


\begin{minted}{julia}
f(x) = x^2
\end{minted}

However, you \textbf{can not} have at-blocks, docstrings, doctests etc. in an admonition.

Headings are OK though:

\chapter{Heading 1}

\section{Heading 2}

\subsection{Heading 3}

\subsubsection{Heading 4}

\paragraph{Heading 5}

\subparagraph{Heading 6}

\end{quote}


\hypertarget{1173158838627777693}{}


\subparagraph{Info admonition}



\begin{quote}
\textbf{{\textquotesingle}info{\textquotesingle} admonition}

This is a \texttt{!!! info}-type admonition. This is the same as a \texttt{!!! note}-type.

\end{quote}

\section{Lists}



Tight lists look as follows



\begin{itemize}
\item Lorem ipsum dolor sit amet, consectetur adipiscing elit.


\item Nulla quis venenatis justo.


\item In non \emph{sodales} eros.

\end{itemize}


If the lists contain paragraphs or other block level elements, they look like this:



\begin{itemize}
\item Morbi et varius nisl, eu semper orci.

Donec vel nibh sapien. Maecenas ultricies mauris sapien. Nunc et sem ac justo ultricies dignissim ac vitae sem.

Fusce tempus scelerisque egestas. Pellentesque varius nulla a varius fringilla.

\begin{itemize}

\section{Doctesting example}



Often you want to write code example such as this:




\begin{minted}{jlcon}
julia> f(x) = x^2
f (generic function with 1 method)

julia> f(3)
9
\end{minted}



If you write them as a \texttt{```jldoctest} code block, Documenter can make sure that the doctest has not become outdated. See \href{@ref}{Doctests} for more information.



Script-style doctests are supported too:




\begin{minted}{julia}
2 + 2
# output
4
\end{minted}

\section{Running interactive code}



\href{@ref}{\texttt{@example} block} run a code snippet and insert the output into the document. E.g. the following Markdown




\begin{lstlisting}
```@example
2 + 3
```
\end{lstlisting}



becomes the following code-output block pair




\begin{minted}{julia}
2 + 3
\end{minted}


\begin{lstlisting}
5
\end{lstlisting}



If the last element can be rendered as \texttt{text/latex}:




\begin{minted}{julia}
struct LaTeXRender
    s :: String
end
function Base.show(io, ::MIME"text/latex", r::LaTeXRender)
    write(io, """
    Rendering \texttt{LaTeXRender}: $(r.s)
    """)
end
\end{minted}




\begin{minted}{julia}
LaTeXRender("render this")
\end{minted}

Rendering 	exttt{LaTeXRender}: render this


If the last value in an \texttt{@example} block is a \texttt{nothing}, the standard output from the blocks{\textquotesingle} evaluation gets displayed instead




\begin{minted}{julia}
println("Hello World")
\end{minted}


\begin{lstlisting}
Hello World
\end{lstlisting}



However, do note that if the block prints to standard output, but also has a final non-\texttt{nothing} value, the standard output just gets discarded:




\begin{minted}{julia}
println("Hello World")
42
\end{minted}


\begin{lstlisting}
42
\end{lstlisting}

\subsection{Color output}



Output from \href{@ref}{\texttt{@repl} block}s and \href{@ref}{\texttt{@example} block}s support colored output, tranforming ANSI color codes to HTML.



\hypertarget{2007526222437001005}{}


\subsubsection{Colored \texttt{@repl} block output}



\textbf{Input:}




\begin{lstlisting}
```@repl
printstyled("This should be in bold light cyan.", color=:light_cyan, bold=true)
```
\end{lstlisting}



\textbf{Output:}




\begin{minted}{jlcon}
julia> printstyled("This should be in bold cyan.", color=:cyan, bold=true)
This should be in bold cyan.
\end{minted}



\textbf{Locally disabled color:}




\begin{lstlisting}
```@repl; ansicolor=false
printstyled("This should be in bold light cyan.", color=:light_cyan, bold=true)
```
\end{lstlisting}




\begin{minted}{jlcon}
julia> printstyled("This should be in bold light cyan.", color=:light_cyan, bold=true)
This should be in bold light cyan.
\end{minted}

\end{minted}
